Uit welke elementen bestaat elke accordion section in de DOM-tree?

De accordeon section bestaat uit een geïntegreerd html document; dit begint met een Doctype=html.
Hierin werd een head-element en body-element geplaatst.
In het body-element werd een div-element geplaatst, met daarin een aantal heading-elementen.
Naast de head-elementen zit er ook nog een Sectienaam, en een span-element in genest.
In ditzelfde body-element werd er een nieuwe div aangemaakt met daarin een p-element met de eigenlijke tekst die we te zien krijgen.

Welke wijzigingen gebeuren er in de DOM-tree als je op een gesloten section klikt?

Nadat er wordt geklikt op een andere sectie in de accordion section verandert er voornamelijk iets in de body.
Bij de heading veranderen alle booleans naar false, alles wat op active stond verandert naar collapsed en tot slot zal de tabindex van 0 naar -1 veranderen.
Bij de aangeklikte sectie, diegene die zal worden uitgeklapt, gebeurt het omgekeerde.

Heel wat attributen toegevoegd aan de oorspronkelijke elementen. Hoe komt dit?

Bij de Chrome developer tools elementen werd er bij de "h3" en "div" elementen steeds een “class” aangemaakt, wat de accordeon specifieerde.
Bij de view source code werd er enkel een ID toegevoegd.
